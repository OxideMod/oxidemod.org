  6. ohgodwhydidiregister

    Koraka said: ↑
    So there seems to be an issue with HumanNPC and Waypoints. The NPC doesn't pause when you set a 2 waypoints at the same location. I looked at both of the plugins code, and it looks like the movement is handled by HumanNPC and not Waypoints. Is there any chance someone can look into why NPCs don't pause movement please?

    Couple other things. NPCs can't seem to draw LOS to a player past a barricade or sandbag. They always seem to goto a wounded state, and when killed don't give say anything in message_kill if they have a message_hurt saved.

    I fixed the no pausing issue.
    Original Code (Line 333)
    Code:
    if (npc.player.transform.position == wp.Position)
                {
                    npc.DisableMove();
                    npc.Invoke("AllowMove", GetSpeed(wp.Speed));
                    return;
                }
    Modified Code
    Code:
    if (Vector3.Distance(npc.player.transform.position, wp.Position) < 0.01)
                {
                    npc.DisableMove();
                    npc.Invoke("AllowMove", GetSpeed(wp.Speed));
                    return;
                }
    After some extensive google-fu I found that the operator == for Vector3 is only true if they are with in 1e-5 (0.00001) of each other. When a waypoint is saved it is stored with it stores the x value with 3 decimal points, y value with 7 decimal points, and the z value with 4 decimal points. Whenever I saved 2 waypoints at the same spot, the z value is always 0.0001 off from the previous. This meant that they would never be == with each other.

    So instead I changed the if conditional to check the distance between the 2 points and if they are within 0.01 of each other then it allows them to pause. I don't know if this will be a big enough distance separation or if it will cause issues in other parts of the code. But for now it seems to work.

    Still working on the other issues.
