  14. Twilight42

    I've done some tests desperately hoping it will help fix the bug because I really like this mod.
    Apparently it seems the server crash is associated with the Defend option. Human NPC will work perfectly if defend is set to false. We can kill/shoot do whatever and it won't crash server.

    However if defend is true and human player attacks NPC, it will crash server immediately.
    All below tested with latest oxide build & mod. Freshly created dedicated server. Without any further Mods except Kits/Pathfinding/Waypoint/HumanNPC

    Custom kit applied with guns

    Defend false & Attacked by player : Dies. No issue.
    Defend false & Attacked by animal : Dies. No issue.
    Defend true & Attacked by player : Crash!
    Defend true & Attacked by animal : Shoots animal to death. No issue. (THIS IS FUNNY...it only crashes when attacked by humans)

    Hope it helps fix the issue.

    Oop sorry I didn't notice it . If you follow the trace of the method call, OnAttacked just does a death check and invokes the player being injured on the ground.
