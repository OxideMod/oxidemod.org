    Odd... I have done this so many times...

    Ok.. Open the Plugin Data directory (oxide/data) and open/edit the HumanNPC.json ... Take a look at this line:
    "HumanNPCs": [
    {
    "userid": 1995233227,
    "displayName": "Guard",
    "invulnerability": false,
    "health": 50.0,
    "respawn": true,
    "respawnSeconds": 30.0,
    "spawnInfo": {
    "position": "78.7495 17.33399 -229.2438",
    "rotation": "-0.115105 0.5371411 -0.07428588 -0.8322932"
    },
    "waypoint": null,
    "collisionRadius": 1000.0,
    --->>> "spawnkit": "sniper", <-----

    See if it reflects the kit you added.... Keep in mind userid is the ID of the NPC (in case you have more then one).

    You can manually change it to "kit name" if it isn't.

    If none of this works....

    F1
    oxide.unload HumanNPC
    oxide.load HumanNPC

  2. DesupermannO

    XSlevinn said: ↑
    So you'll need to have items in your inventory, do /kit add name to set the kit. Then when you do that, you set your options such as npconly true, etc. when you do that make sure you type in items to copy your items from your inventory to your kit. Then do the /npc kit command. I was having the same problem as you til I realized I wasn't typing in items after doing /kit add name

    so for example:

    >items in inventory<
    open chat
    /kit add bowman
    >hit enter to send<
    open chat
    /kit authlevel 2 hide true npconly true items
    >enter to send<
