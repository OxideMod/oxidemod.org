    but it really seems like it's a huge amount of work for little, but i'll still look into it and see
    [DOUBLEPOST=1427716383,1427665574][/DOUBLEPOST]just a simple explaination:
    when you attack, all the informations are created on the client side (so this is NOT accessible via the server side)
    and managed by the server side.
    But in the client side there is already a lot of lines of code (more then 100 lines, that we CANT access or reproduce as it's not present on the server side) just to make the attack and send it to the server
    but then on the server side everything is handled also, bullets are created, damage is dealt, etc it's more then 200lines if you have to take in count creating melee or distance damage (creating the arrow that will got up and down, or the bullets) and then you have to trace the arrow to the target and then deal damage (which is even more code)
    and in some of this code some are not accessible directly as they are protected and we would need a work around to use those stuff.

    so yes everything nearly is possible, but it would require a LOT of code for a very small improvement, and there is a lot of chance that on every update the code gets broken because they changed 1 line or 1 function or even rewrote most of it.

    That's why i try to reproduce as good as i can without having to go in all this coding as it would just make the plugin very unstable, and very hard to keep up to date.
    If rust was finished, it could probably be more usefull to get into doing such a code, but at the moment it's risky.

    currently what i'm trying to do is to create a pathfinding for the NPC, so they have a more logical way of moving, i'm not really good enough, but maybe with some help we will manage to make something nice, that won't be able to break between rust updates as it would be a personal code.
