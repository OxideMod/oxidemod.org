    It takes time for the issue to occur and if you have people on the server doing rad towns, you most likely won't see it much. Basically, the spawner seems to not check if there is something anymore before refreshing boxes so once you have enough boxes and it should stop spawning them, it just keeps spawning them infinitely. If you let an empty server run for a week I bet you'd have dozens of boxes stacked on top of each other.

    I highly recommend using that fix, as you can guess it can easily throw balance off. I have a test server nobody plays on and I've seen as many as 7 boxes on top of each other, I'm fairly sure there is no cap to this.

    If you're using the last version, the default config has everything at off except fixloot which is set to true (this is what you want in your case). Only thing you might want to do is decrease the cycle time (it's 15 min by default), I'd go for 300 seconds (5 min) in your case as with everything off, it won't be very hungry performance-wise.
    [DOUBLEPOST=1439309288][/DOUBLEPOST]Edit: The fixloot function does not handle barrels, that documentation is wrong. It only handles the box crates and weapon crates.
