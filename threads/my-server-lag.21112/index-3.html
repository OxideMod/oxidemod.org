    I think there should be a big sticky post about this to be honest, as it's not a simple answer. Here are some things to start looking into though, there are plenty more than this.

    - Make sure your server hardware/network is strong enough to handle the number of people you're allowing.
    - Make sure your map isn't too massive for your hardware, 4000(is most popular), more enities = more memory and can lag when loading/saving.
    - Also you should experiment with your server save interval (via server.saveinterval) by possibly increasing it and ideally announcing it to your users so they know if the server is lagging during that time that's why.
    - You should set a fps limit(via fps.limit), maybe try 120
    - Restart the server at least once a day(ideally twice), this is a *HUGE thing and pretty essential for any popular server.
    - Make sure you start your server with -nographics
    - You should look into plugins that help keep entity count down, such as Auto Purge for Rust | Oxide NoDecay for Rust | Oxide ( increasing twig decay to remove crappy raid bases ) maybe remove bases with no cupboards and or loot after x amount of time.maybe in your rules limit the number of high external layers players can have. Set reasonable gather amounts with Gather Manager for Rust | Oxide and drop loot via BetterLoot for Rust | Oxide. Auto kick players with high ping. If you run events, set a reasonable player entry count.
    - Understand every plugin you have running, one or multiple may be the culprit.
    - If you're using a VPS see if you can save up for a dedicated server(ideally with a fast single core, since I read recently rust mostly uses a single core)
    - If anything else resource intense is running on your rust server see if you can move it(ts server, database etc...)
    - Check out this guide Rust server performance – EU BEST and google for others.
