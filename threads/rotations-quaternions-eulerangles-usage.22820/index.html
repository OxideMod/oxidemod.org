  2. EinTime

    Hey Colon I don't think I quite understand. What do you mean by "references the same" or "reference offset"? Unity uses local rotations for all transforms. So the rotation of any object is always thought of as "in reference to" the rotation matrix of it's parent. You are trying to rotate a parent entity, do you want the children to rotate with their parent (you rotate parent 30 degrees in the X axis, children all rotate in reference to that?) or do you want the children to not rotate while the parent does rotate.

    For the first option you dont need to touch the children, just rotate the parent transform and the children will follow as you would expect. For the second option you can either call a transform.LookAt() on the children after you rotate the parent, or set their transform.rotation in some other way.

    Is this what you mean? Can you give an example or code example?

  5. k1lly0u

    Code:
                void Awake()
            {
                entity = GetComponent<BaseEntity>();
            
                string prefabattached = "assets/prefabs/deployable/furnace/furnace.prefab";
                attachedentity = GameManager.server.CreateEntity(prefabattached, entity.transform.position);
                attachedentity?.Spawn();

                attachedentity.SetParent(entity);
                attachedentity.transform.localEulerAngles = new Vector3(90, 0, 0);
                attachedentity.transform.localPosition = new Vector3(1.55f, -0.40f, 2f);
            }
    By setting the parent you can manipulate its position and rotation locally using the parents transform as its base
