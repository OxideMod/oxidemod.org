    Hi there.
    Ill input my Advise, I mentioned details about this in the page before.
    This IS a rust issue not a Oxide issue, after various tests it seems to be related to the speed and angle that the bullet is traveling at before hitting the ground, any shots fired at a high angle have caused this issue.
    If it spams RCON I have noticed Rusty will lag and even crash as it can not keep up, But I have had no issues with the server itself.
    The solution for Rcon is to add it to the filter from console list.

    All mods cause some kind of resource use, normally small though, but if something is used often that can affect it, for example if you have "Timed Execute" running commands every minute that's resources being used.
    But I feel most of the issues are caused by High Pop.
    I say this as there have been many forum discussions about lag on high pop servers, I dont know if this is because rust itself is not designed well for that many players or if the server program/server is poorly designed.
    As I dont see this error often but I have noticed High pop servers do, I would say that this is just a side effect issue of the lag caused by high pop.
    This means that I feel if you fixed what caused this error it would not help reduce the lag. the best thing you can do is ask Rust to improve the performance on servers with 100+ pop
