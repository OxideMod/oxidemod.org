    private object OnItemCraft(ItemCraftTask task, BasePlayer crafter)
        {
            var itemname = task.blueprint.targetItem.displayName.english;
            if (AdminInstantCraft && task.owner.net.connection.authLevel == 2) task.endTime = 1f;
            if (ModeratorInstantCraft && task.owner.net.connection.authLevel == 1) task.endTime = 1f;

            task.cancelled = true;
            SendChatMessage(crafter, CraftBlockedItem, itemname);
            foreach (var amount in task.blueprint.ingredients)
                crafter.inventory.GiveItem(amount.itemid, (int)amount.amount * task.amount, false);

            return false;
        }

  6. Deicide666ra

    Visa, in the original code you posted the problem is that the blueprint you get there is the actual game-definition blueprint. When you subtract 10 seconds from it there, you are subtracting 10 seconds from the blueprint time for ALL players on the server. Next time you get in that function with the same item, you'll get the SAME blueprint reference and if you keep subtracting 10 sec every time you'll get to zero time and eventually negative time, etc.

    If you look at LimeBench (a crafting mod I did) you'll see that when the plugin starts, I store all the blueprint times into a dictionary. Each time the OnIemCraft hook is called, I always compute the crafting time from those values I stored... the blueprint time itself is unreliable because I keep changing it depending on who is crafting the said item.

    Also, when the plugin unloads, I set back the crafting times for all BP's to what they were on start... This ensures that changes my plugin did are undone when it unloads.. Otherwise reloading the plugin could result in some blueprints times getting multiple reductions ,etc.
