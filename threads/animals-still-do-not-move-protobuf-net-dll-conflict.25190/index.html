  11. ^Animal^

    Here is the reply from my server support.
    I am sure most server company's are doing the same to keep the servers running.
    My server is through Ketzen.com

    Hey Animal,

    Thanks for getting in touch, it's nice to hear from you.

    At the moment, we have Animal AI turned off across all Rust Servers as the technology behind the pathfinding (navmesh) that was introduced earlier in the month has had serious implications on server CPU usage - this means that when a server is loading up, it will use 100% of all CPU cores for the entire duration of the server boot, which is roughly 5 minutes. When said server is loaded up, it will typically use 5x more CPU than it has historically or with navmesh turned off.

    As this is the case, we took the decision, along with many other providers, to disable navmesh from the last update until Facepunch and Unity can improve the performance on it, as currently as it stands, navmesh freezes other servers and the physical machine if more than 1 Rust servers loads up at the same time.

    The good news is that we are seeing improved performance with navmesh day-to-day on the staging development branch, and will enable it across the board once it is feasible to have enabled within a shared server environment. Our tests this week have shown a markable improvement on boot up, however servers are still using roughly 40% more resources when in use with navmesh enabled.

    It's not something we took the decision to do lightly and was made rather than increasing the cost that we can provide Rust servers at, apologies that no notification was sent - it should have been sent from the Game Panel automatically when the update was pushed out.

    I hope this clears things up.

    Kind regards,
    Alasdair
