  2. Express

    For the record, I found that pretty much all of this was caused by running an 'oversized' map, as per this parameter:
    +server.worldsize 4000

    I had it set to the maximum, 8000... upon creating a new server a couple months after this I used the default size of 4000 and everything was fine.

    I'm not trying to gravedig this thread, but I guess that others may also encounter this problem, and may find this information useful.

  3. AlienX

    I have this problem as well, I've put it down to problems between the servers and clients terrain height map generation where the server thinks some terrain is slightly higher than the clients, thus the server attempts to fix your position but gravity on your client pulls you back down to the ground again.

    The server again attempts to compensate and pulls you back up, giving the impression that your on a really weird bouncy castle - this goes round and round in a loop until you move to terrain that is considered the same height by the server.

    I did find that the glitchy area's got better with time though.
