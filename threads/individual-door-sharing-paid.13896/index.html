    We have had a alot of requests from players in regards to doing individual door share rather than the current door share plugin which is sharing all doors.

    Sharing all doors does not work when you build community based buildings and want to control the codes on such a building.

    My suggestion would be
    #Re-write doorshare in C#
    # /share all "Name"
    # /share door "Name"
    # /share door remove "Name"
    # /share all remove "Name"
    # /authlist (While looking at a door prints a list of people who have access)(Will only return for the entity owner or if authorized on the door)
    # /share clear (removes all shares you have ever made)
    # Make sure it has dictionary / name lookup and can support entry by steamid
    # Entity Owner integration for door ownership confirmation
    # Make sure admins can use the /share door "Name" and bypass the entity owner check
    # Add config to allow for admin masterkey to be used on a specific authevel which can be modified.
    # Add config to allow / disallow share all
    # Do not allow door share to work with key locks
    # Permissions for door share commands share.all share.door share.admin

    This would cover the share all and the share individual to allow people to really be creative with some of the play styles and buildings they are working with and maybe create areas where their trusted friends can come and go where the owner has full control over the list at all times rather than needing to give access to the whole base.
