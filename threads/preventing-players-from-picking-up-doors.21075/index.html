  7. TeamRequem

    Not sure if this would work out for you, but a work-around is to put a lock on the door and make sure to return true to CanOpenDoor. The player will be prompted to unlock the door, thus can't remove it but will still be able to freely open the door without knowing the code. It's how I've achieved this in a future game-mode to be released.

    Works for hatches as well.

            if (!isTooFar(player.transform.position))
                return true;
            return (!door.IsLocked() || door.HasLockPermission(player));
        }
