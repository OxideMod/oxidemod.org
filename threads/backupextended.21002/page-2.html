    Fujikura updated Backup Extended with a new update entry:

    0.2.0

    • Changed backup processing
    • removed startup backup functions ( read below )
    • added ResourceID
    To be full compatible with the internal backup processing on server startup it does backup all custom folders above "3" to the next one (if exist).
    This free's up folder number "3" which then will/could be overwritten on server startup backups.     This ensures full data consistency.
