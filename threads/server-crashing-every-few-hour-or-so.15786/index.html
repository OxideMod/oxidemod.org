  2. HelloKitty

    for me i had a issue with a server crashing when someone or me removed building parts.
    Seemed to be a corrupt save file because when i reverted back to a older savefile a few ours earlier the problem was gone.

    Do it crash with ANY kind of plugin added? or does it crash with a specific plugin?

    i would do list of things before i redo it all but hard to sitt here and do a megapost on all kinds of things .

    but..
    (i will write this as if you are almost a nub but i am not assuming you are a idiot because of it ).
    if at all possible after clients approval i would do a complete restart of the "project" for this client.
    As in i would writecfg, save, quit (in that order) and hope it closes propperly. or if already crashed .

    Then backup whole folder. just rename the steamfolder that you use to install it.
    Then redo all my starter steps with downloading steamcmd, check so the folder i pick to install the steamcmd and the appcache files is a completely fresh with new names different from before.
    (when i wipe my server i go from c:\xxx1\server1\steamcmd.exe to c:\xxx2\server2\steamcmd.exe to c:\xxx3\server3\steamcmd.exe and so on after each wipe generation).

    then i install with the bat i use in that new specific folder :
    Code:
    @C:\xxx2\server2\steamcmd.exe +login anonymous +force_install_dir C:\xxx2\server2\rustdedicated\ +app_update 258550 validate +quit
pause
    Then i download a new zip of the newest oxide, and then newest rustio version.
    thus completely 100% fresh install

    Now it has no plugins however besides rustio (wich is a plugin i doubt is causing this when server is not overloaded).

    What i would do then is redownload manually every single plugin from the websites FRESH..
    and adding them one by one.
    But since it is "randomly" crashing it is hard to say what specific plugin it is since you cant spend 5 hours playing with one plugin each .
    But so far you have no old datafolder and no old config files (config files i would prefer to config when the plugins is added. incase there is something added in them that actually breaks a plugin that magicaly breaks the server).
    You also have no old save file and userpercistance.db.
    And if he used signature artist then he also has no storage.db file.

    If the server crashes after adding the plugins then one or more of the plugins is causing the crash and you have a guessing game on your hand trying to find what plugin it is.
    Any plugin that adds entitys or manipulates ingame entitys or manipulates player BPs or inventory and such is more likely to cause a corrupt savefile, storagefile or userpercistance.

    If the server runs flawless with all his lovely plugins.
    Then i would copy+paste the backed up save file and userpercistance
    Adding plugin data is something that requires a FULL shutdown, same with storage.db for pictures on signs and savefile.

    if it is still live and well after you restored all this then you have a fixed server and we are all happy and cheerfull.
    If not, then one of the added files has caused it, thus corrupt save file, or corrupt storage.db (i had corrupt storage.db aswell but it only crashed specific clients, not whole server).

    The next step is to try a older .sav file (0 is newest, 9 is oldest. and hope for it to not be all 10 old and corrupt saves).

    I hope this helps and was understandable . its harder to give a quick guide then doing it myself.
