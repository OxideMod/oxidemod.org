  10. Kanivara

    So interesting bug cropped up on my server....

    When someone starts a vote for the time, a vote for the weather can simultaneously be started. The vote goes through as the weather vote, the time vote is overruled completely. This breaks the plugin and requires a restart of the server.

    Is there any way to prevent a weather vote and a time vote from occurring simultaneously?

    ETA: I figured out exactly what breaks. When the weather vote is started while a time vote is being done, the weather vote takes all the vote input that the time vote had received and, upon completion, prevents the time vote cooldown from starting because the time vote never completed.
