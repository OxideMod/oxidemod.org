  2. sqroot

    Anything to reduce the entity count.
    Slow down the server gameplay by reducing gathering rates, speed up decay, wipe more regularly, stuff like that.
    I've found that Rust's decay iteration can cause lots of lag and this is primarily caused by a huge entity count.
    The same goes for the garbage collector iteration roughly every three minutes. This doesn't just slow down the server, it even halts it for a couple of seconds. Entities are one thing which affect this, a large player pop alone can increase the duration of the pause by a lot as well, though.

    I've also noticed an incredible amount of page misses caused by Rust. Not sure if disabling swap actually helps here, I've never tested it, but theoretically it should help a great deal with lag (only do this as long as you've got lots of RAM left, though, because otherwise things will start to crash when your RAM limit is reached!).

    If you're using Windows, disable power saving mode. This can bump up your server performance by a good 20%.
