    Hello, I had the same problem, defined the area and configured to not be able to build or destroy but still did not work. Now my configuration is managing to do it. The ZoneManager.json file put in the "\save\rust\oxidize\data" and ZoneManager.cs file in the "\save\rust\oxidize\plugins."

    To create an area where you can not build:
    /zone_edit XXXXX (available in Zonemanager.json)

    Then do (what to do will now be for the chosen area)
    /zone nobuild true
    /zone nodeploy true
    /zone radius XX
    /zone enter_message "Message" (opçional)
    /zone leave_message "Mesage" (opçional)

    If you are logged in rcon, you can place normally, but the player may not deploy buildings in the defined area.

    This configuration is optimal for radioactive areas, such as the Small-Radtown. So no one can build in this area, thus avoiding lag to the server.

    ===============
    To create a safe zone, where you can not kill the players who are sleeping, for example. You can set a warp point for this area.

    /zone_add NAME
    /zone_edit XXX (not the name, but the ID, available in ZoneManager.json (date))

    then just do:
    /zone sleepgod true
    /zone pvpgod true
    /zone nobuild true
    /zone nodeploy true
    /zone enter_message "Message" (optional)
    /zone leave_message "Mesage" (optional)
    /zone radius XX
