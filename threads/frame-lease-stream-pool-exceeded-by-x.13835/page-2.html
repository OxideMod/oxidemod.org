    Your structureframpacketlimit is half of default. Default is 200. No reason to have it lower as this will make it take longer for people to load into server and for structures to render. Increase it to reduce load in times and structure rendering times. Put it too high, however, and you will see lag when players join server.

    Here is my autoexec

    -exec host 12871;queryport 12881;loadmod 717024931;loadmod 704019543;loadmod 717255124;loadmod 744760222;loadmod 708232676;maxplayers 50;servername <color=#E85900>AZTEC WARFARE</color> FRESH WIPE 9/21
    loadbalancerframebudget 400
    combatlogtimeout 30
    afkkicktime 3600
    addadmin 76561198101068839
    addadmin 76561198091588048
    vehicledecaytime 21600
    autosaveenabled 1
    structuredecayenabled 1
    structuredecayfrequency 1200
    structuredecaydamage 30 60
    autobackupinterval 7200
    spawnercellupdatesperframe 10
    structureframepacketlimit 400
    quit 86400


    Note that I have made modifications to things like afkkick and vehicledecaytime so that's why that is different than yours, and I changed the way decay works (it is still the same strength and time in the end, just updates less frequently to reduce instances of lags when it does).

    Also currently my spawnercellupdatesperframe is default, I just keep forgetting to take that out XD (i was experimenting with it and default is best).

    I also highly recommend getting rid of the ping command and replacing it with the ping universal mod, so players can simply type /ping to see their ping. If you find that players are being kicked too much upon loading into your server I can instruct you how to set a delay on the ping detection to give people more of a chance to load in before kick. This is another problem with the ping command, it scans for ping every second so if your ping happens to be over 300 while loading in.... you'll be kicked.... but often times ping approaches that number naturally upon load in. But first thing is first, let's get your server running good before worrying about ping limits XD

Instead get your server using /loc via location mod and triangulating it on hurtworldmap.com. If you already have installed this you must shut down server and delete its files and then restart in order to uninstall.

    - Don't use Sleepers mod

    - Don't use magic chat

    - Always check oxide/logs/most recent log file (NOT compiler) upon server loadup for mod errors
