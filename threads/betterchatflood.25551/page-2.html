    This has a conflict with BetterChat 5.0.12 / BetterChatFilter 1.2. If a user spams a censored word fast enough, it is uncensored, shows up in chat (ignoring the BetterChat filter) and does not have an error message. This ONLY happens if BetterChatFlood is reloaded BEFORE BetterChatFilter is.

    TL;DR:
    To create a plugin conflict, first load BetterChatFlood, then BetterChatFilter

    To alleviate the plugin conflict is first to reload BetterChatFilter
    Then, reload BetterChatFlood

    When fixing the conflict, you have to wait several seconds in between the reloads, or else the conflict will continue again. I've not been able to consistently reproduce this (but I don't have any more time to at the moment)

    Just trying to help out any players receiving plugin conflicts, or perhaps if the devs can learn more about this conflict and somehow get it resolved.
