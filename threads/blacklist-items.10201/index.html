    The rarity would require a server restart, since existing barrels and crates containing these items would not be refreshed by just reloading the plugin. When you restart the server all barrels and radtown crates are destroyed and slowly recreated after the restart. I honestly never tested this method of blocking drops, but as far as I know items with a rarity of "None" should not be spawned by the game. You might have to use LootConfig if they still do.

    As for the research again I never used it, but normally if UserCraftable is set to false, I assume the server would prevent players from crafting them. It might require a server restart, I'm unsure. It might only apply for users after they relog as well, I'm not sure.
