    The plugin doesn't distinguish damage types and just checks if the player that is taking damage has god mode enabled, and if this is the case it will replace the damage that will be dealt to 0. The attacker can be a player, an animal, environmental, suicide (kill command) or even the helicopter. It will always block the damage.

    If the plugin is running, no errors are showing but functionality seems to be iffy as you describe then it is possible another plugin you are running is interfering with the godmode plugin because the plugin still blocks all damage as it has been doing previously.

    Merged thread here as it sounds like you are trying to use this plugin, not Rust's godmode feature. To use it, you need to be set as owner as mentioned above. To use Oxide's permission commands, you also need to be set as owner.

    Once you have your owner permissions set for your server, you can use the command "grant group admin cangodmode" in your console. You can also use this command via RCON without the owner auth level set.
