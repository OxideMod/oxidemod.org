  10. Zarkazm

    I dont personally use the stashes, so i have no personal preference on how i'd "want" it to work, but being in a software development background, im looking at this objectivaly.

    The first issue is that stashes do not have a concept of decay, but rather follow a "despawn" principle. So, the first step is to add configurable options to adjust the despawn delay and to disable despawning entirely.

    The next task is adding decay to stashes. However, this brings us to a big part that currently this plugin doesnt do, which is setting the delay time. The question being "what should be the delay time before decay starts to occur?". The best answer would probably be to default it to 3 days, like the despawning, and make it configurable. But for consistency sake, you open it up for people to want the delay time configurable on everything.

    Another question would be the default damage scale. Same as Twigs or Wood maybe? And of course, making the scale configurable like the plugin is already designed to do.

    Hope this viewpoint helps. I guess the question for you is "how far do you as the developer want to take it"?. As far as making a separate plugin, thats more for your sanity. As a user, one plugin with a config file is sufficient enough for me personally. I suppose making a separate "NoDespawn" or "NoDelay" plugin makes some sense, but adding those features into NoDecay, making them configurable, and turning them off by default would be just as simple for a user i think.
     
    Zarkazm, May 29, 2016
