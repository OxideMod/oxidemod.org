  4. FuJiCuRa

    Zombie_Fate said: ↑
    A trick i'm not aware of?
    Click to expand...
    The current logic of config handling is that way, that it does revert the config file to the defaults, when the file has wrong formatting or wrong syntax.
    Simple syntax errors can be a comma too much or to less. Thats enough to let the config reset to its defaults, to provide a failback.
    There's nothing what can be changed there, until i would move the GUI-pages-config into a single data file. That way i'm sure the plugin would tell you a loading error on syntax problems, but would not revert it to zero.
