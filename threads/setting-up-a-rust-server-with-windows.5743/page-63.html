  4. BAT.Kraydon

    I use gameservers too.

    you can manually install oxide 2 quite easily by over writing the files in the rust server directories with the new oxide files (some are new files that will just paste in without overwriting anything).
    Always best to take a backup of any files it overwrites just in case but with gameservers you can always do a re-install if needed (although this wipes everything including bp's)
    and yes all done via FTP (i use filezilla)
    Also the Oxide directory is created by oxide to host the plugins etc. this isn't the oxide 2 files, they go in the rust server folders as above. you can see the directories in the downloaded oxide 2 file where they need to go.

    To restore mods etc you only need to download them from the plugins, config, and data folder plus also the users, bans and server files if you have used those.
    Then you can restore them to the new plugins, config and data folder when they are re-created.
    The main issue you could have though is the plugins you will restore are not updated and so you need to download the up to date ones. I usually do all this in steps otherwise rushing into getting mods installed can cause you more problems. So do the rust update, wait till the oxide 2 is out then install that (manually or via gameservers install (although this doesnt have a version against it grrr and can take a while to update to the latest oxide 2 version)) then download the latest compatible mods as they come out and update with your data files accordingly (although some data files can change so sometimes these also need manually updating instead of just copying across).

    Seems a headache but actually not that bad once you do it once or twice.

  6. BAT.Kraydon

    Hi,

    Thought i'd re-write it out a bit better instead of the above. This could work for other server hosts too but might need slight modifications to the below instructions. Also don't blame me if it doesn't work

    So backup the following directories (or files within)
    cfg folder (contains bans.cfg, server.cfg, users.cfg)
    oxide/config folder (contains plugin config and data files)
    oxide/data folder (contains plugin data files)
    oxide/plugins folder (contains plugin files)
    (can just backup the whole oxide folder if you wish but i dont bother)

    backup any files that the new oxide 2 install will overwrite within the ruste and RustDedicated_data/managed folders

    login to gameservers portal and on MODS tab install server update

    If oxide 2 is updated on gameservers (good luck in finding out though) then you can also install oxide 2 via the MODS tab.

    Otherwise download oxide 2 update and upload via ftp the latest oxide 2 files to the correct directories on the server.
    If the oxide 2 directory isnt showing, then after a server restart it will show up.
    If the oxide 2 directory was already there it's probably best to clear out the config, data and plugins folder otherwise you could get lots of errors until the mods are updated and installed.

    Download any updated mods and update configs accordingly although these might have to be done manually as some mods change their structure and you could mess them up by just copying the files across.

    Oxide 2 hasn't been updated yet and then the hosts probably won't update till a while after it's out so if you want it straight after oxide 2 being updated you will most likely need to manually install, unless you can wait for Nitrous to update.

    Yes there is a good chance the configs will be overwritten but if you have a backup you can manually update the new config with the old information. Check the new config and data files first because if they are the same layout then you can most likely just copy those back across but if not then you will need to manually update them to the new layout otherwise the mod can break.
