  12. Vedmaka 2

    Hi, Guys!

    I've encountered problem i can not solve for already a day or so, maybe you will help me somehow.
    Problem is simple: i have a zone marked with undestr flag, so buildings receive not damage in this zone. One day i run with Incendiary bullets there i've noticed that players receive no damage from flames produced via this bullets. Same thing with Incendiary rocket for Rocket Launcher - players receive no damage from flames last after shot.

    After some investigation i've figured out that reason of that is on the line 776 - when damaged entity is not a BasePlayer and not an NPC this block executed, so damage gets cancelled. When i comments out "CancelDamage" call players start to receive damage from flames, but buildings obviously starting receive damage too. With this line uncommented - neither player or building receive damage from flames.

    Code:
                    HashSet<Zone> zones;
                if (!buildingZones.TryGetValue(entity, out zones)) return;
                foreach (var zone in zones)
                {
                    if (HasZoneFlag(zone, ZoneFlags.UnDestr))
                    {
                        CancelDamage(hitinfo);
                        break;
                    }
                }
    Considering this i concluded that flames damage somehow tightened with building damage - is seems like mystic: I've created separate hook at other plugin for OnEntityTakeDamage and added some simple output to it:

    Code:
    void OnEntityTakeDamage(BaseCombatEntity entity, HitInfo hitinfo)
        {
            if (entity == null) return;
            if (entity is BasePlayer)
            {
                Puts("Player takes damage");
                Puts( hitinfo.damageTypes.GetMajorityDamageType().ToString());
                Puts(hitinfo.damageTypes.Total().ToString() );
            }
        }
    When i remove undestr flag from zone and standing in flames this code reports for some damage amount and damage type = Heat. When i put undestr flag back this code reports about 0 damage value and type = ElectricShock.

    So, here is a mystery:

    * When undestr flag removed - OnEntityTakeDamage called in ZoneManager plugin falls into "not player" condition and cancels damage to BuildingBlock, but, SOMEHOW this affects future player damage from flames, it becomes 0 too.

    Help me, i am missing something !
