    That code won't fix your issue - that fix only impacts issues where players are affected by a zone outside of the zone.

    But if you want, you can try my other suggestion which is for buildings only - modifying the procedures OnTriggerEnter and OnTriggerExit, in ZoneManager.cs (between lines 485-530 approximately), and adding a check to make sure the building entity is in the zone:
    Code:
    // in OnTriggerEnter
                    foreach (var instance in colliders.Values)
                        if(instance.collider.bounds.Intersects(gameObject.GetComponent<Collider>().bounds)) // added this
                            CheckCollisionEnter(instance.collider);
...
// in OnTriggerExit
                    foreach (var instance in colliders.Values)
                        if(instance.collider.bounds.Intersects(gameObject.GetComponent<Collider>().bounds)) // added this
                            CheckCollisionLeave(instance.collider);
    Keep in mind, there are no null checks for the colliders, so it's possible it could throw an error (although I've not seen any).
