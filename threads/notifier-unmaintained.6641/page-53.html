    Click to expand...
    Hide Admins setting will hide all the admins anyway on all lists. In example, even if you have /admins working, normal players wont see any admins listed at all, Notifier will tell them there are no admins online, even if they actually are (think of this a way to trick cheaters). Besides this, Hide Admins will not let admins connection messages, and will make sure admins names will have the default player color on all messages as well.

    Alright guys, I know I'm taking much more time than I told you, but it's for a good reason. In only 10 mins or so I did all this so far for the next version:

    - Added Enable Players Default Colors setting, whether or not force player names colors to be the game's default, otherwise they can be colored any color from each message.
    - Added Boards Title to Colors, to change the title color of all the boards to any color.
    - Added exception to log messages when the path is not found, fixing common errors.
    - Removed all the temporary code from previous versions.
    - Fixed creating commands even if they were disabled.
    - Connection and Welcome messages use the players default cache names again. (i.e its back to {username} instead of {user} or {player})
    - It's now possible to use any of the players cache info on Welcome Message. ({username}, {steamid}, {country}, {auth}, {ip})
