    It's not all about convenience, as it often ends up with one plugin breaking every other plugin. Oxide is made to be modular, hence the whole usage of plugins, having an all encompassing plugin defeats the purpose of even using Oxide's plugin system. By having one giant plugin to cover all your bases, all it takes is one typo for your whole system to come crumbling down, leaving your server with nothing. So when you end up scrambling for plugins to replace it, as the developer has since disappeared. You find your admin tools, protection methods, etc unavailable, as the one godzilla plugin provided them all. Want to use a plugin that provides one feature better? You'd have to either hack up the plugin or find replacements to each feature.

    So yes, while it may be convenient, it's not exactly smart. Plugin developers come and go in the gaming community, so flexibility is important to most. It plays right along with the saying "don't put all the eggs in one basket."

    There's definitely no harm in learning either, just keep that in mind when building your server and developing plugins.
