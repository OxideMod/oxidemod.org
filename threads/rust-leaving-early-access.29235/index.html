  12. Wulf Community Admin

    Generally we wouldn't maintain support for a development branch as it would cause more work for us, especially if they updates are frequently. It also creates an issue where users are bugging plugin developers to update for something that is likely to change the next day, which would also cause issues with compatibility (in most cases) with the released version of the game they support as well. While we did add support for the itemv2 branch of Hurtworld in addition to the public/release branch, updates to the itemv2 branch are not daily. Trying to allow for plugins to work on both is also a complication, as it involves some additional changes on both Oxide's part and each plugin developer that wants to update for that.

    So tl;dr, generally not worth the time and effort, especially since we aren't paid to sit around an update for every little development change.
     
    Wulf, Jan 29, 2018
