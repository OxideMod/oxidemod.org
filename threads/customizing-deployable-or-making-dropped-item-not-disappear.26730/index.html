  1. Vlad-00003

    Ok, so let's say I have a tool cupboard and I want it to look different from the over ones. The simple way - skin - wouldn't work here, as the tool cupboards doesn't have a skins.
    So I was thinking about creating some worlditem on top of it. It was easy - this way was already discussed somewhere on the forum - so, here is a simplifyed code:
    Code:
    private Dictionary<uint, DroppedItem> entites = new Dictionary<uint, DroppedItem>();
void OnHammerHit(BasePlayer player, HitInfo info)
{
    if (!entites.ContainsKey(entity.net.ID))
    {
        DroppedItem item = ItemManager.CreateByItemID(-1224598842, 1, 962868059)
            .Drop(entity.transform.position + Vector3.up * 2, Vector3.zero, entity.transform.rotation)
            .GetComponent<DroppedItem>();
        item.GetComponent<Rigidbody>().isKinematic = true;
        item.GetComponent<Rigidbody>().useGravity = false;
        item.allowPickup = false;
        entites.Add(entity.net.ID, item);
        return;
    }
    entites[entity.net.ID].Kill();
    entites.Remove(entity.net.ID);
}
    And everything is cool, the item spawns etc, but.... after server restart it just drops to the ground. And after some time (server.itemdespawn) it would dissaper like a normal item.
    So - my question is here - How can I make them stay, or is there any over way to make it looks different?
    P.S. And please - don't tell me to add supply signal smoke effeect to it. It looks wierd and this cups would be easily detected by over players =)

  2. DylanSMR

    Code:
    item.CancelInvoke("IdleDestroy");
    should work to fix the despawn time. When it drops on server reset, I assume that is because the server is just recreating the item, and not retaining its rigidbody. So you would probably have to remove it on unload, and spawn them on Loaded(with some data file saving the ones needing to be respawned over server restart)
