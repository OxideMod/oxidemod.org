  15. aardvarlk

    The error has to do with the banner image. Here are the steps to fix it. Working on AdminPanel v1.2.8, AdminRadar v4.2.1, Vanish 0.5.1, and GodMode v4.1.1.

    Open adminpanel.cs with any text editor.

    Add // to the start of lines 94 and 496.
    Code:
    // cacheImage();
    Code:
    // new CuiRawImageComponent { Color = "1.00 1.00 1.00 1", Png = fetchImage("AdminPanalImage"), FadeIn = 0 },
    This is all that's needed to make the plugin run, but it's still out of date and incompatible with the others.

    To fix AdminRadar compatibility, change line 82.
    Code:
    AdminRadar.Call("cmdESP", player, "radar", new string[0]);
    The UI doesn't update properly when disabling radar, but at least it toggles. [EDIT: Fix below]

    To fix Vanish compatibility, change line 443:
    Code:
    if (Vanish && permission.UserHasPermission(player.UserIDString, "vanish.use"))
    To fix GodMode compatibility, change line 466:
    Code:
    if (Godmode && permission.UserHasPermission(player.UserIDString, "godmode.toggle"))
    Edit: Fixed the AdminRadar button not changing colour correctly. This is a messy change, because it should actually probably be fixed on AdminRadar's end. Also, I don't actually know C#, but this works well enough...

    Change lines 42, 64:
    Code:
    AdminGui(player, false);
    Change line 156, 212, 252:
    Code:
    AdminGui(player, IsRadar(player.UserIDString));
    Change line 82 (yes, again):
    Code:
    var wasRadar = IsRadar(player.UserIDString); AdminRadar.Call("cmdESP", player, "radar", new string[0]);
    Change line 83:
    Code:
    AdminGui(player, !wasRadar);
    Change line 357:
    Code:
    private void AdminGui(BasePlayer player, bool isRadar)
    Change line 368:
    Code:
    if (AdminRadar) { if (isRadar) { BTNColorRadar = btnActColor; }; };
