  5. sqroot

    I'll shamefully try to extend some earlier answers.

    1) Oxide uses /plugins for the actual plugin files, /lang for localization (some plugins allow you to configure translations for messages, users can also choose which language they want to use with a command), /data for storing persistent plugin data that needs to be transferred across server restarts and finally /cfg for configuring the plugin.
    You can delete the plugin without deleting anything in the other folders - it isn't necessary to delete them. It only makes sense to delete the respective files in those other folders if you want to keep it overseeable, or some data files are too large for your storage.

    2) No, it isn't. Oxide is a server sided mod, meaning everything it does happens on the server. The server doesn't know about the textures the client uses and never communicates with the client for textures, so it's impossible to change textures via the server side.

    3) It isn't, because item designs are client sided and item functionality is both client and server sided. Because it isn't possible to add client side functionality for a specific entity or a new item design to the client from the server, it isn't possible to do this.

    4) This is possible, take a look at Zones Manager for Rust | Oxide. The options undestr, nobuild, nodecay, nodeploy, noremove, noupgrade and possibly nostability might be of interest to you.

    5) Firstly, not every update is a wipe update. Map wipes (usually) occur on the first update of every month. Map wipes exist because the respective updates usually break compatability with the map or certain items, meaning the old map file (which contains all inventories, the map and obviously all items on the map as well) will not be compatible with the new update anymore. If you are very careful and have made sure that what you're trying to transfer across map wipes is still compatible with the new version (meaning you read the patch notes very carefully, this includes items in loot containers copied!), then you can use a plugin to transfer the building. Copy-Paste for Rust | Oxide may work for this.

    6) CustomLootSpawns for Rust | Oxide may work for this.

    7) Don't take everything on the forums at face value. Test it yourself before jumping to any conclusions, many server owners reporting lag issues aren't properly singling out the issue.
