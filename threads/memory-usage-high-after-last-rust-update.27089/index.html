  2. Hider

    Solved:
    if you have opened the server for a long time and you have never done this, but you want to offload the server's memory and processor a lot, then simply remove the oxide / data / oxide.users.data
    My oxide.users.data weighed about 30mb, which accumulated for 8 months and when loading with this ballast the server consumed just over 4GB of memory.
    As far as I understand, this file stores the database, assigning to each new player a group of oxide.
    After the removal, the server began to consume 1700-2300 mb (tested by the different sizes of maps)
    I think this will greatly affect the performance of the server and will be useful for admins if you did not already knew
