    I've had a server up and running for about a week now and farmed a ton of items. The only thing that I changed was the multiplier (now set to 2x). It seems that Goat frames and Roach Engines of any flavor are not spawning. Is there a way to fix this? LootConfig is the only loot plugin that I have on the server.
    Click to expand...
    Editing LootConfig would not effect whether or not cars spawn. The default values for car spawns are quite low and each car can take hours to spawn. After you restart the server or reload the plugin the car spawns will go back to the minimum spawn level (unless more people have claimed cars than the minimum) , which I believe is 6 or something like that. So it is likely the cars just haven't spawned yet - either that or there is a lot of claimed cars on your server which, without increasing the maximum spawn numbers, will result in less cars spawning in rad towns as the car spawner counts both claimed and unclaimed cars, even though it doesn't delete claimed cars when resetting to the minimum.

  9. TruxDeadnitro

    go to your folder or control panel where your server is and then filemanager oxide/config/LootConfig,json Download the file and change the
    Code:
    "GlobalStackSizeMultiplier": 1.0,
    to what ever you want so for example 500.0.
    Then get back inside of your server and use /reload LootConfig
    When done that your server is X500.
    the X500 is just for example you can change it to every X that you want
    And to install the plugin go to oxide/plugin and upload it on there now you can do two things restart your server.
    Or go inside of your server and use the same thing /reload LootConfig
