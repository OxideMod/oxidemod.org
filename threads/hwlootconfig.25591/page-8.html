  10. wuzziefuzzy

    i think u mean how to config it?.
    first connect to filezilla,
    then go to this path: /Servername/oxide/config
    next go to the file LootConfig.json and open it with notepad c++.
    Then at the top off the file you will see this line "GlobalStackSizeMultiplier": 1.0,
    Change the 1.0 to 3.0 or whatever u like to multiplie the loot. :"GlobalStackSizeMultiplier": 3.0,
    Next go in the server and type /reload LootConfig.
    And ur done I hope this is what u mean

    "Key": 0.03,
    "Value": {
    "RollCount": 1,
    "RollWithoutReplacement": true,
    "LootResult": {
    "ItemId": "DetonatorCap",
    "StackSize": 1,
    "RandomVariance": 0

    Change the Key value to something you like, that is the drop rate off the item i hope heheh

  13. Ionstorm

    Hi this plugin is working. However every time you reload the Addon it over-writes the config file you edited and starts it from scratch. This is obviously done so as if any new items are introduced with new server versions they are included by default in to the config..

    The actual flexibility of increasing drop chances etc is fab, but the needing new config files, especially when no new items are introduced is annoying as hell, flexible but hell )

    PS. Tried to Read Only the config file, plugin throws a bitch fit! Not a good idea


    TEMP WORK AROUND

    Change your LootConfig.json backup file to have these Details:

    Code:
    "Version": 23,
"VersionConfig": 3,
    This will allow it to load an older config file and save you the pain of having to constantly re-create the plugin. Be aware when items are added, this won't work, as you'll not get the new items dropping.

    It's a temporary fix for a lot of people.

    IMPORTANT: Keep a backup of your "LootConfig.json" because for some reason when the plugin creates a new one and renames the existing one to LootConfig.json.old it removes a crapload of code from it (becomes a smaller file size and items such as Det Cap are removed). Just found that out the hard way!

    If you're dedicated copy your tweaked LootConfig.json on to your desktop, If you're shared keep a copy of it on your desktop and copy + paste to the online editor // upload via FTP.


    PS: Did anyone else notice the new config file is setting 255 of items to drop? Someone just got 255 paint from one box

    for me the LootConfig.json and LootConfig.json.old have the identical size (tried by changing the version)
    on every protocol version change it will be created an backup of your current config and fresh values will be exported from the game, so that you can compare the changes, merge them and reapply your config...(best is to use WinMerge or some other diff tool to compare these two files)
    if you don't care about new or changed loot you can simply recover the LootConfig.json.old to LootConfig.json and update the "Version" to the latest one and you are done...
