    Keep in mind that it will update Rust but wipe Oxide out at the same time. If you have mods (or players) that are sensitive to that, you might want to not autoupdate and do both manually at the same time. In most cases a modless server is better than a server nobody can join because it's not been updated, but depending on the mods it's not always true.

    Most monthly updates include procgen changes that can slightly or dramatically change a map. A majority of time, it's impossible to get the same map you had... In this context, not wiping is pretty much impossible. There is at least one plugin out there that copy/pastes bases, so I suppose someone really motivated could expand from it and make a full world save of the items, sleepers, their inventory content, all building blocks, crates and their contents, etc. but that would be a massive undertaking.

    One easy thing that could be done is something I saw on a server on legacy. They saved player inventories so that after a wipe you'd still have your full inventory. People would load up on stuff before wipe and start with that after, on a wiped map + wiped BP's....
