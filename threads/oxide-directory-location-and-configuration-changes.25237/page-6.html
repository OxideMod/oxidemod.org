    If you're on a Windows dedi, my post a couple of days ago should help as it looks like I got it right first time. If you're on a Linux dedi it should still help.

    Oxide directory location and configuration changes | Page 2 | Oxide

    I can confirm that if on Windows and you use something like this as your command line to start up Rust, it will be as if the Oxide change didn't happen and everything will remain where it was. In other words the Rust server will start and use the directory \server\Server01\ as the location of all your game files.

    Code:
    RustDedicated.exe -batchmode +server.identity Server01 +oxide.directory "\server\{server.identity}\oxide" +server.port 28015 +rcon.port 28016 +rcon.password "########"
