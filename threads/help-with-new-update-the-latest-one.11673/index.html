  14. NobodyFTW

    Either using server.url in the command line or in the config files its the same thing, the config files will only be loaded if their specified on the command line as well, so it's pretty much the same thing. I would only recommend the config file in case you are running a bunch of commands through the command line. Though if you follow my example you can just do this once and the url will be the same everytime, and you may change it whenever you feel like it.

    Code:
    -batchmode +server.port 28015 +server.hostname "SkinN # Tests Server" +rcon.password "XXX" +server.seed 0 +server.worldsize 1500 +server.url "https://www.google.com/"
