  6. Tekoud

    On the plugin page it is explained Death Notes for Rust | Oxide .
    So in the top you see 2 permissions that are possible with the mod, one of them says:
    deathnotes.see needed to see the messages (if enabled in the config file)

    So they tell you the following (Do this):
    grant group player permissionname

    So if the permission is deathnotes.see the line would become:
    grant group player deathnotes.see

    Type that in your console.

    Then type server.writecfg or the setting is gone after restart.

    So what people ask of us is that we learn to understand the permission system, when doing it a few times it'll be easy.

    In real short for me this works, so you got 2 ways of addmin permissions, grant group player permissionname and grant group admin permissionname.
    So when i know that i only need the permission name from each mod and assign it this way to either the player group or admin group.
    That's all that's to it to understand the basics and to activate permissions when needed for player or admin.

    A backup of your permissions is in \oxide\data oxide.groups.data.
