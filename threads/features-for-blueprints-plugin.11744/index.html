    Well there is one very important feature missing from all blueprint plugins that's been requested several times and that I was about to write something for since nobody seems to want to take the bait on it but since you're proposing

    The request was to be able to do partial BP wipes. An admin could for example have a list of "non-wipe" blueprints (or the other way around, a list of bp's that should wipe) or alternatively work by tier (wipe all BP's from the two top tiers for instance).

    This would allow admins to do partial BP wipes instead of full wipes. Players would lose access to high tier guns and raiding tools, but retain the ability to make basic clothing/weapons/large box, etc.

    All the tools to do this exist, it's just a matter of putting it together and having a reliable mechanism to process this and not double-wipe or forget to wipe a player, etc.

    The main issue here though is that there is no "RemoveBlueprint" functionality in Rust, so you basically need to make a list of what BP's the player has, wipe them and then re-add what should be carried on.

    Also, I don't know if this can be done while players are offline? If not, it would complexify the whole process a bit, you'd need to process player login and only wipe him at that moment IF he hasn't been wiped before.

    I've been wanting that for my own server for a while, just never got around to it but it was pretty much next on my list. If you want to take it over, be my guest, I have more than enough project ideas on my plate
