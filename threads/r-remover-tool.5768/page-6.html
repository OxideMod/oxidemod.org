    The ownerid and moderatorid commands can only be used directly in the server console or in a rcon client that connects to the server using the rcon port & password (rusty or another rcon client). Using one of these commands in the in-game console (a.k.a. developer console now) does not work at all. Basically FPS removed the rcon.login command for admins, and they're now saved to users.cfg. So everytime a player that was added with ownerid or moderatorid to that list will have authlevel 2 or 1 respectively which results in access to the admin commands. After using the ownerid or moderatorid you'll have to reconnect to your server if you were already in-game and you should notice in the server console/rcon client that when you login it should now say "<player> has auth level <1 or 2>".

    Another option would be editing the users.cfg file in the cfg folder but normally the commands should do the trick.
