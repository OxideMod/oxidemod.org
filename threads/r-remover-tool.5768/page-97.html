  12. Nocturnalverse

    Code:
      "Remove - Access - Require Block Ownership": true,
  "Remove - Access - Use Friends & Require Ownership (Require Block Ownership must be true)": true,
  "Remove - Access - Use ToolCupboards": false,
    Edit: This is an excerpt from RemoverTool.json. You can find it in your server's Oxide/config directory.

    Use this with Friends API and you have a greifless remove tool. This works beautifully because it doesn't care about cupboard access. Not being able to remove cupboards themselves is a small price to pay for being able to implement this tool and not have entire bases /removed after a raid. Werkat said he is working on the cupboard issue. If two players are /friend add with each other they can remove each other's deployables/structures. It's super fucking simple. The Friends API even has a working Code lock and Turret share feature as far as I can tell, plus it is separate from RustIO, so you can have 2 different levels of trust. If this isn't working for you guys there is some user error involved or some idiosyncratic thing going on with your server/host and Werkrat isn't going to be able to fix that kind of thing.
