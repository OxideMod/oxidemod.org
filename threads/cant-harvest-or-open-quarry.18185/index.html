    [DOUBLEPOST=1463919129][/DOUBLEPOST]I have solved our problem.
    It seems that something was amiss after the update last Thursday to our previous setup and possibly not a plugin error.

    What I did to resolve the problem;

    1, Created a new folder, just copied over my .bat launch file and added the line;
    2, C:\steamcmd\steamcmd.exe +login anonymous +force_install_dir c:\rustserver\ +app_update 258550 +quit
    3, All server files where newly downloaded and installed.
    4, Launch the server, once finished loading, logged in and tested quarry, all ok so saved and closed the server.
    5, Copied over the Oxide files and removed the c:\steamcmd....... line from my launch.bat file
    6, Launch the server, once finished loading, logged in and tested quarry, all ok so saved and closed the server.
    7, Copied over the RustIO file
    8, Launch the server, once finished loading, logged in and tested quarry, all ok so saved and closed the server.
    9, Copied over just my plugins folder
    10,Launch the server, once finished loading, logged in and tested quarry, all ok so saved and closed the server.
    11, Copied over my Config folder
    12, Launch the server, once finished loading, logged in and tested quarry, all ok so saved and closed the server.
    13, Copied over my whole ServerName folder, made necessary changes to launch.bat file
    14, Perfect, still had all my map data, player data, BP's and the Quarry's now work.

    I'm still unsure as to what caused this issue but i think i will follow this procedure on every Thursday from now on, minus the copying of the map and user data on a fresh wipe.

    Hope this helps you.
