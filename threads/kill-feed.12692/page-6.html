    So the Kill Feed uses the hook CanBeWounded(BasePlayer player, HitInfo info) to detect if a new player related entry should be displayed. The hook is called each time a specific section of code on the server is executed. That section I'm referring to is executed if the player is about to die and is not already in the wounded state. It determines if he should be wounded or if the wounding state should be skipped. Regardless of the result it might display a new entry, given some other factors are right, as well.

    It really doesn't display kills at all, so to speak. Practically. sometimes if the wounded state is skipped it seems like it displays the kill, but that depends entirely on the result of that specific code section the hook is called in.

    The name Kill Feed might be misleading here, because the feed doesn't tell you if you killed someone, but then again, Wounded Feed sounds shit...
