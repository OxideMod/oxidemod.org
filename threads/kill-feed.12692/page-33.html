    Keep in mind that eventually, when Facepunch gets around to fixing their stuff, it will be better to set 3.1 use server file storage to true again. Using the server file storage is way better because once the client (player) receives an item icon image he will actually cache it locally so he can just reuse it instantly when it's needed the next time around. If you don't use the server file storage the icon image will be accessed (downloaded) over the URL every single time it is needed.
