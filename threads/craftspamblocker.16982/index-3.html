  1. Wulf Community Admin

    LeoCurtss submitted a new resource:

    Craft Spam Blocker - Prevents items from being crafted if the player's inventory is full

    If a player's inventory is full, the item is not crafted and a message is displayed in chat.

    At this time, resources are still consumed when they attempt to craft with a full inventory.

    This is meant to help prevent players from spamming the server with dropped crafted items if their inventory is full.

    Thanks to CCTV and RHAKOON for requesting the plugin!

  15. sqroot

    Here's a version that's a lot more agressive.
    Please note the following: This will cancel *all* current tasks without refund if a task finishes while the inventory is full and cancel only the newly added task with a refund when a new task is added while the inventory is full.
    This is to avoid spam via refunds (dropping refunds might lag the server as well) and to avoid spam clicking.
    There's a nicer and more user friendly way to do this (requeueing the crafting task and gradually increasing the crafting time, but it'd take more effort and I'll maybe have a look into it some other time), but you can use this for the mean time.
    Keep that in mind. Increasing the crafting time for low cost items via CraftingController might be a way better and nicer solution for your players than this!
     
    sqroot, Jul 12, 2016
