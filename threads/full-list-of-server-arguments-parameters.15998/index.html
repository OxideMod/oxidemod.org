  5. sqroot

    +server.identity?
    Several different configurations (which includes a lot more that just a single associated cfg file for the server) for the same server are managed by their identity.
    If you want a smaller difference between instances than what the identity provides, you'd probably have to write something that manages this yourself, although it's probably not a good idea.
    Everything within a server identity can be written to (except for unused saves and oxide plugins), so using a single identity from multiple instances would cause trouble.

    So in the end, here is what i got :
    Code:
    # Server settings
# More settings available after install in serverfiles/server/rust-server/server.cfg
servername="Rust LGSM Server"
ip="0.0.0.0"
port="28015"
rconport="28016"
rconpassword="CHANGE_ME" # Don't let it blank but change it
maxplayers="50"# Advanced
worldsize="4000" # default 4000; min : 2000 max : 8000
saveinterval="300" # Auto-save in seconds
tickrate="30" # default 30; min acceptable 15 , max 100 or 128# https://developer.valvesoftware.com/wiki/Rust_Dedicated_Server
fn_parms(){
parms="-batchmode +server.ip ${ip} +server.port ${port} +server.tickrate ${tickrate} +server.hostname \"${servername}\" +server.identity \"${servicename}\" +server.maxplayers ${maxplayers} +server.worldsize ${worldsize} +server.saveinterval ${saveinterval} +rcon.ip ${ip} +rcon.port ${rconport} +rcon.password \"${rconpassword}\"  -logfile ${gamelogfile}"
}# Folders and logfile name to get 1 logfile per reboot
# Logging
logdays="7"
gamelogdir="${rootdir}/log/server"
scriptlogdir="${rootdir}/log/script"
consolelogdir="${rootdir}/log/console"gamelog="${gamelogdir}/${servicename}-game.log"
scriptlog="${scriptlogdir}/${servicename}-script.log"
consolelog="${consolelogdir}/${servicename}-console.log"
emaillog="${scriptlogdir}/${servicename}-email.log"
gamelogfile="\"gamelog-$(date '+%Y-%m-%d-%H-%M-%S').log\""
# There is also a function to move gamelog files to a proper log folder on every restart ;)
    So, RAW start parameters with default values (you should set some of them, like ips, otherwise if you got several interfaces it will listen on the first eth interface that you got for example) are :
    Code:
    -batchmode +server.ip 0.0.0.0 +server.port 28015 +server.tickrate 30 +server.hostname "Your Server Name" +server.identity "your_server_identity" +server.maxplayers 50 +server.worldsize 4000 +server.saveinterval 600 +rcon.ip 0.0.0.0 +rcon.port 28016 +rcon.password "Your_Rcon_Password" -logfile logfilename.log"

    Explanation :
    -batchmode (to run with no interface=
    +server.ip 0.0.0.0 (to set your listening IP
    +server.port 28015 (to set your game and query port)
    +server.tickrate 30 (default 30, can be from 15 to 100 or 128, but 30 is OK)
    +server.hostname "Your Server Name" (No comment)
    +server.identity "your_server_identity" (Used mainly for multiple instances, and that's nice to have a custom folder name, in server/your_server_identity . If you change it, your server is kind of wiped unless you put the "old" identity)
    +server.maxplayers 50 (self explanatory)
    +server.worldsize 4000 (default 4000, min 2000, max 8000)
    +server.saveinterval 600 (time in seconds for server save, default 600 -10 minutes-)
    +rcon.ip 0.0.0.0 (listening IP for RCON)
    +rcon.port 28016 (listening port for RCON
    +rcon.password "Your_Rcon_Password" (RCON password, if you leave it blank after setting it up once, the server won't start, btw you shouldn leave it blank ever)
    -logfile logfilename.log" (logfile name, if your're scripting, add the date to it, so it doesn't get erased at every server restart)
