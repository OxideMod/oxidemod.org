  10. iDeath

    @NaJunHui Yes, huge problem right now. Steam updated the server culling API and is now affecting fail over IPs, particularly OVH's since they route with a /32 network mask through their dhcp server.

    There is a temporary fix by post routing your machines IP with your failover IP on Linux, haven't tested on Windows.

    Linux
    Code:
    iptables -t nat -A POSTROUTING --destination 0.0.0.0/0 -j SNAT --to <FAILOVERIP>
    Confirm entry
    Code:
    iptables -t nat -nv -L POSTROUTING
    I am not familiar with the netsh interface to give you accurate information about post routing.
     
    iDeath, Feb 8, 2018
