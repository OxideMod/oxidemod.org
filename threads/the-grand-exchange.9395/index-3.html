  12. Scorpyon

    Scorpyon updated The Grand Exchange with a new update entry:

    Bug Fix - /setprice

    - The /setprice command should now set prices properly and be reflected in the store list
    Click to expand...
    Read the rest of this update entry...
    [DOUBLEPOST=1434187186][/DOUBLEPOST]"Bug Report: When you look at the numbers next to the items in the plugin they are weird. "

    The reason for this is that the prices stored in the plugin itself are increased by a factor of 1000. This is so that I can easily modify prices on a percentage basis without them being messed up by integers forcing values to 0, etc.
    So all the items that are currently in the defaultList are stored ni this way to make incremental changes to their price.
    It also stops crazy things happening when people buy resources in stacks of 1000!! (Which was a problem I found! Heh!)
     
    Scorpyon, Jun 13, 2015
