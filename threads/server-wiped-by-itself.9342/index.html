    The Save plugins do not corrupt your saves, they use the same method that Rust uses for saving. Other plugins that manipulate the world or entities could potentially corrupt the save though, but there are no known plugins that do that at this time. More than likely the map is getting bugged from Rust itself, or it just isn't being loaded properly.

    Tips: make sure -load is used in the command-line or startup script. If using the HapisIsland map, also make sure to specify a server.seed, else it creates a new map each time the server is restarted.
