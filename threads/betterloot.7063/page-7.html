  5. dcode

    Alright guys, I think I found the issue. You are using a config for ItemConfig which is for version 1.0.8 of the plugin, but your plugin version is lower, in SannyOK's case 1.0.7. The config seems to have changed significantly between these two versions. More precisely, item rarities are no longer stored as an integer but as a string, resulting in rarities of all game items to be set to "None", which in turn breaks BetterLoot.

    TL;DR: All your items and blueprints are messed up! Update ItemConfig to the latest version and make sure that the config file you use is actually compatible with it!
     
    Last edited by a moderator: Mar 20, 2015
