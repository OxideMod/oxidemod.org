    BetterLoot updates the barrels after the barrel has already spawned, using the hook OnEntitySpawned, which is the same hook ExplodingBarrels uses, so there's likely a conflict there (BetterLoot updating barrels after ExplodingBarrels has already checked them). You could try renaming the hook in ExplodingBarrels and then calling it from BetterLoot after it has run through PopulateContainer and finished adding the new items to the container... But that's a theoretical solution I've not tested.
