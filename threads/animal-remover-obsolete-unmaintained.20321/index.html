    did you try to debug the OnEntitySpawn ?
    Because last time i tried to use your way, the server kept spamming trying to spawn the animals.
    It made my server go nuts.

    If i could give you a suggestion, it would be to directly change it from the source of the spawnables.

    Code:
     var f = typeof(SpawnPopulation).GetField("Prefabs", (BindingFlags.NonPublic | BindingFlags.Static | BindingFlags.Instance));
            foreach(var s in Resources.FindObjectsOfTypeAll<Spawnable>())
            {
                if (s.Population == null) continue;
                if ( s.Population.ResourceFolder.Contains("animals"))
                {
                    Debug.Log(s.Population.ResourceFolder);
                    Debug.Log(s.Population.LookupFileName());
                    var p = f.GetValue(s.Population) as Prefab<Spawnable>[];
                    foreach(var pr in p)
                    {
                        Debug.Log(pr.Name.ToString());
                    }
                }
               
            }
    It will show you what the server likes to spawn. Just remove some Prefab<Spawnable> directly from there and it should be much more light weight and you won't need any hooks.
