  2. Wulf Community Admin

    The oxide.users.data file only stores what is told to store. If a user's name changes the next time they connect, it's updated in that file. If a user is assigned a permission from an Oxide command or method, it's updated in that file. It's unlikely that the problem is with the file, but instead the plugin that is adding benefits to the user.
