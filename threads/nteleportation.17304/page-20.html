    Nogrod, First I must say thanks for the plugin. I always run into these quite important unsupported plugins , and spend days trying to fix them. I guess some people lose interest in Rust, or gaming in general, or life takes over, and they just stop updating their plugins.

    Besides the one issue I have this wipe with NoDecay and the Survival Fish Trap being added, I have one other issue, possibly with this plugin.
    When I paste a structure with the copy/paste plugin, I can't set home in the pasted structure.
    First I use prod plugin, and it will say "no owner found for this building block".
    Then I will use building owners plugin /changeowner (ME), and it successfully gives me the structure.
    At this point the Nteleportation plugin still cant set home. It says "You or a friend need to own the house to use home!".
    To fix this I remove one of the foundations and replace it. Now setting home works, but only on the foundation that I removed and replaced.

    So people pasting on water or wherever they can't install a foundation cant set home. That is good. I think people should only paste where building is legit.

    So for me it works as long as I replace a foundation. I don't want to remove the owner check, or foundation check of this plugin. We need them for proper balance, but somewhere between prod, building owners, copy paste, and this plugin there is an issue. I'm thinking that pasting never links the structure with the ground, and even if prod tells you that you own the structure, something in Rust or Nteleport still knows the building is not anchored to the ground.

    This is not urgent. copy paste is a VIP thing, and the fix as I stated could easily be implemented each time a server pastes a structure for a VIP.
    Put this on the back burner, and thanks again for rescuing so many unsupported, yet important plugins.
