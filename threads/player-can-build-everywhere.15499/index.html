  9. Calytic Community Admin Community Mod

    To be clear,

    V2.0.0 fixed the pillar exploit but reintroduced an exploit that was previously fixed. This exploit is not easy to discover or accomplish.

    V2.0.2 of EntityOwner fixes both exploits.

    It is unfortunate you reverted.

    The first exploit, or pillar exploit, allowed players to remove foundations or pillars they did not build (when useToolCupboard: false). As I said, this was fixed with 2.0.0.

    The second exploit (stuck privilege) is due how quickly EntityOwner 2.0.0 attributes ownership to Tool Cupboards when deploying them with Removal Tool activated. This was only possible with version 2.0.0.

    V0.1.61 did not attribute ownership to Tool Cupboards, thus negating the second exploit, but still it has the first.

    On a side note: The same bug that caused the stuck privilege exploit also causes a stuck building blocked bug, thus necessitating admins to use "authclean" a whole lot. As I mentioned, this bug was also fixed in version 2.0.2.
