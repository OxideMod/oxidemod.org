    When a server update is released then an Oxide update is required but in general plugins do not need to be deleted and usually still work without any issues unless there where major changes to the server's code in the update. At the point where the server update is released it is usually better to wait to update your server until the Oxide update is released for the new server update because using an older Oxide build will result in downgrading some of your server files again which will either cause the server not to run or behave erratically. How you update your server depends on how your server is hosted.

    When you host your server on your own machine or a dedicated server with remote access and you have to run all steps manually you would need to run steamcmd to download the updated server files and then download the latest oxide release, basically following the installation steps again.

    When you have a gameserver with one of the available GSPs then you usually only have access to a web-based control panel which should have the options to update the server using steamcmd and normally also to install oxide. I can't really provide a lot of info on this part as this varies per GSP, but there should at least be the options that I have mentioned somewhere.
