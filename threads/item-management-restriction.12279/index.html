  3. Deicide666ra

    To the OP, blocking an item altogether is much easier said than done unfortunately. Item Config and LootConfig for example will allow this, but their configuration needs to be redone each time a new update to Rust is released and it's a relatively complex file to toy with. Works well for simple changes but if you want to make some significant changes to the loot tables, IMHO, they don't really cut it anymore unless you can program and make your own parser to process the config file (been there, done that).

    You might want to look at BetterLoot... It has it's flaws, but it has a blacklist of items. You can also check Crafting Controller which can be used to prevent some items from being crafted altogether (ItemConfig will allow that as well if I remember right).

    Bottom line is there isn't anything currently that does exactly what you need easily.

  9. Deicide666ra

    Just to expand on that.. Client and server versions are paired and totally incompatible with higher or lower versions. Since the game is on Steam, most player's clients are automatically updated when a new version comes out. If you opt to not update your server, you will end up with no players in aa very short time.

    Players cannot revert their client versions or have more than 1 installed..

    The solution is to use a plugin that handles this in a way that works through updates. Not saying it's impossible, it just requires a plugin made with that goal in mind. ItemConfig is basically just a plugin that dumps the server data to a file and when you reload it it just reloads that data and updates the server with it. Since new items can be added, removed, change names, properties can be added, etc. The author opted to just ditch the config on updates.

    I suppose it could be modified to attempt a merge between the old and new version but thats not how its done for now.
     
    Deicide666ra, Oct 15, 2015

  15. Calytic Community Admin Community Mod

    you could write a mod which iterates through all the entities in the game and spits out their asset name. keep in mind that on a server with a lot of entities, this code will cause substantial lag until the loop completes.

    Code:
    GameObject[] gameobjects = UnityEngine.Object.FindObjectsOfType<GameObject>();
foreach (GameObject go in gameobjects)
{
      Debug.Log(go.name)
}
