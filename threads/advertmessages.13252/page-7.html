  7. TermNet

    I think there's an issue where the last message in the list isn't sent, and I think it has to do with the random.Next implementation already doing a "-1" internally.

    According to MSDN:

    MSDN https://msdn.microsoft.com/en-us/library/2dx6wyd4(v=vs.110).aspx said:
    The Next(Int32, Int32) overload returns random integers that range from minValue to maxValue – 1.
    Click to expand...
    So with line 35:

    Code:
    do advert = random.Next(0, Configuration.Messages.Count - 1);
    Should be

    Code:
    do advert = random.Next(0, Configuration.Messages.Count);
