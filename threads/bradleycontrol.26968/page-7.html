    Wasn't the config problem... First problem is that bradley respawns every X minutes (which is set in config), no matter if it was killed or not. Second problem, is that in the case, if it wasn't killed, its aren't removing from the List<BradleyAPC> bradleys, so, after few respawns (bradleyAverage setting), it stops respawning at all, even by server event. Third problem is that it will not respawn by the plugin if its dead. Instead you have to wait until server will respawn it for you.

    So that setting means RESETdelay, not respawn.
