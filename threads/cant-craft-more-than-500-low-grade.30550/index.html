    @tomhud It's common for small things like this to go unnoticed mainly for the reason of the very few who know about it aren't willing to tell anyone to avoid it being patched. I wouldn't say this is a Unity issue either I think it's more so just an optimization issue for Rust in general.

    In short, the exploit was when someone queued multiple stacks and canceled them all at the same time. This causes the server to refund all of the resources for every single stack and without these queue limits, players would be able to do it as many times as they want to. Eventually, it'll stall the server and for most of them with potato hosting it'd result in a crash almost every time. The same thing with the old map exploit, it really only affected servers on potatoes.
