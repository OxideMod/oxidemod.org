  12. Gorehammer

    HOW TO FIX YOUR LUSTYMAP:

    OK - we're making some progress. Here's what I've done:
    1. Go into your Oxide folder and delete everything in Storage. (This took 2.5 hours to complete - 25k files and entries - depends on if dedicated server or VM - WARNING: This will remove any custom art you have if you use /sil)
    2. Completely uninstall LustyMap.cs, LustyMap.json, and ALL LustyMap data in the Data folder. Make sure nothing related to LustyMap is anywhere on the server.
    3. Restart the Server
    4. Reinstalled the Lustymap.cs in Oxide > Plugins.
    5. Input your Beancan.io API ID into your LustyMap.json in the Oxide > Config file. Save and transfer back to server.
    6. Allow RUST time to download the auto-generated LustyMap.json (3-5 minutes)
    7. When finished, type out, "resetmap" in RUSTY or the console to force an image deletion and re-download. (another 3-5 minutes)
    8. Wait for RUSTY to tell you that everything was downloaded.
    9. Type into RUSTY "oxide.reload LustyMap"
    10. Disconnect and reconnect to server - now everything works!
    11. 24 hours later it's still working for everyone on my server.
    If the above sequence doesn't work, please repeat steps 6 through 10 a few more times and it should start working. So far I've confirmed 9 servers are fixed from the information above.

    This fixed the issue for me - one of my players still says they see the black map... I'll update if I find anything else.

    The one player I asked to relog and now he can see the map, himself, airdrops, debris, etc...

    So - try the above, and let me know please.
