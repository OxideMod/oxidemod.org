  4. k1lly0u

    The problem likely is that you are resetting the map images when it doesn't work. Just because ImageLibrary has finished storing the files does not necessarily mean the server has finished processing them. By running the resetmap command when it can't find the image is just prolonging the whole process. Its the same with ImageLibrary, users are spamming the refreshallimages when it doesn't work first time and not giving it anytime to finish doing what ever it is doing, and all that comes from it is clearing everything and starting again. If it doesn't work after it says it has finished, wait 5-10 mins and then reload LustyMap. If it still does not work after that then use resetmap. I have never had these issues on either of my dev servers
