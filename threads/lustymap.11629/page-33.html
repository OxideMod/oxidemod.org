    I have had a look at that actually and it should be alot better in the next version.
    I have attached 1.1.11 which i am currently working on if you want to test it

    I have slowed down how frequently the plugin checks for new friends (was previously checking every player against every other player every time the map updates). This is now limited to 10 per second and spread out evenly (once every 100ms). It means that it may take upto 30seconds on a 300 player server to detect new friends and show them on the map but should give a huge boost to performance from before.
    Also i noticed that when the friends was set to off it was still doing most of the work and just not saving the results, now it does the work only if its needed.

    I don't want to post it as an official update yet after having so many issues with the last lot... gonna let things cool off for a while.
