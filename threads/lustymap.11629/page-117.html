    Map Issues:
    I am pretty certain that the issue with the map not working will be solved when FP fix the current problem with signs, which as far as I can tell is related to a issue with a new file storage system. I am reluctant to add a URL option at this time as the amount of time and back tracking required to make this work with a image from a URL is entirely not worth doing at this stage. I have absolutely no desire to rewrite the majority of the plugin for a Rust bug that will probably be fixed soon, however, if this issue has not been resolved in the next 12-16 hours then I will look at a temporary alternative solution

    I haven't spoken to Calytic but I would assume beancan.io has been swamped with requests. There are ~1500 people using the beancan service via this plugin, not to mention all the people that use it outside the scope of this plugin. Either way, at this stage the map download and map splitter will not function correctly anyway
