    It writes to ZoneManager's data because it is ZoneManager creating the zones, this plugin is purely to create all these zones without having to do it manually through zone manager. A perk of this is when you change your map seed it sets up the new zones automatically.
    The plugin will remove all the zones when it is unloaded and will create new ones on server start. It is like this intentionally for a couple of reasons. 1) if you need to edit zonemanager data you can just unload this and then you wont have to sift through all the zone entries this has added. 2) If you decide you dont want to use this plugin, or you only want a couple of zones instead of them all, you just unload, make your changes in the config and reload.
