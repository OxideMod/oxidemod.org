  2. Wulf Community Admin

    1. The command server.writecfg saves what is in the server memory, so if you added commands to the config AFTER starting the server, then it will most likely remove them.
    2. You can add the server.globalchat true in the startup script/command-line, or in server.cfg.
    3. No idea with the tickrate. Either it doesn't exist anymore, or you aren't loading the config.
    4. Poor server performance is generally caused by a config issue or a plugin that interacts with your server in a way such as polling all the structures/objects/etc. Nothing we can really do, as it's up to you to remove/add/test to see what is causing it.
