    Greetings! Any interest in adding rotation parameter input to UpdateZoneDefinition? I've developed a helper plugin to create zones, and had to parameterize rotation in ZoneManager; it might be useful if rotation was accepted as a parameter by the standard version

    I've done the following to accept a rotation around the vertical axis as a parameter (in degrees):

    Code:
    case "rotation":
    float rotation = Convert.ToSingle(args[i + 1]);
    if(rotation != null)
        zone.Rotation = Quaternion.AngleAxis(rotation, Vector3.up).eulerAngles;
    else
    {
        zone.Rotation = player?.GetNetworkRotation() ?? Vector3.zero;/* + Quaternion.AngleAxis(90, Vector3.up).eulerAngles*/
        zone.Rotation.x = 0;
    }
    editvalue = zone.Rotation;
    break;
