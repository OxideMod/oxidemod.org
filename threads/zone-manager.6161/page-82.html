    Works for me:

    ignignokt84 said: ↑
    I believe I've figured out the cause - in method CheckEntities the Physics.OverlapBoxNonAlloc method is passing the full Zone BoxCollider size in, when the method expects HALF size, this causes it to effectively double the size of the zone in all directions from the center.

    In CheckEntities, you can halve the size of the zone. This has solved the issue for me. Can anyone else test this and confirm my results?

    Code:
    {% raw %}
    entities = Physics.OverlapBoxNonAlloc(Info.Location, Info.Size/2f, colBuffer, Quaternion.Euler(Info.Rotation), playersMask);
    {% endraw %}
