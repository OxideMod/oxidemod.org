    So a big thing to note is to facilitate the background customization I had to change some categories of items. Pictures are sorted and saved by category, short name,skinid, then imageID. I say this because if say an item is tagged as a category and yet the picture is in a different category you will get an error. So you may ask how can this happen?

    Well the most likely way is if the item was created prior to the category change or if the data file is manually manipulated(unlikely).

    I'm looking into better error catching to prevent failure and rather just render a question mark or something. I'll make some tweeks in the version I am releasing that fixes the item condition bug.
