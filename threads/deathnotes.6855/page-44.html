  7. CodƎƦ

    Hi guys. You have some bug. When body parts are translated, they still displayed in English. To fix this, please replace this:
    Code:
    if(!raw) bodypart = "<color=" + Config["Colors", "Bodypart"].ToString() + ">" + bodypart + "</color>";
    to this:
    Code:
    if(!raw) bodypart = "<color=" + Config["Colors", "Bodypart"].ToString() + ">" + Config["Bodyparts", bodypart] + "</color>";
    in your function
    Code:
    string GetFormattedBodypart(string bodypart, bool raw)
{
//...
}
    I hope that helped you.

  11. Smoosher

    Disclaimer: Could be wrong im self taught and have never looked at a plugin before for Oxide/Rust.

    The issue seems to be this section of code (line 239)
    Code:
                    if(vic.ToPlayer() == null && hitInfo.Initiator.ToPlayer() == null)
                {
                    return;
                }
    The errors are appearing because animals die of Hunger and Thirst

    When it tries to execute the vic.ToPlayer() and the hitInfo,Intiator.ToPlayer() it NRE's

    I think the hitInfo.Initator is null, so when it trys to run the ToPlayer() method it falls over and implodes and we get the error thrown out in the console.

    So I think i have fixed it by wrapping it and checking to see if it is an animal and if there is a hitInfo.Initator if there isnt it doesnt try and carry on

    My servers not a busy one so its hard to test if ive caused any further issues, but the error for me has now gone, ive uploaded it here if someone wants to give it a bash

    Edit: I also had to handle animal on animal action as that started bleeding through so ive hidden that out
